1. EXECUTIVE SUMMARY & PROJECT GOAL
--------------------------------------------------------------------------------
This project is an end-to-end full-stack Artificial Intelligence & Mobile Computer
Vision system designed to accurately identify photos of Cats and Dogs on an Android
mobile device in real time, while intelligently recognizing and filtering out
non-animal photos (such as human faces, vehicles, furniture, and nature) into a
dedicated "Other (Not Cat/Dog)" category.

The project bridges Python Deep Learning pipeline design (TensorFlow / Keras)
with Mobile Edge Deployment (Android Studio, Kotlin, TensorFlow Lite).

3. DATASET CONSTRUCTION & CURATION (01_data_preparation.py)
--------------------------------------------------------------------------------
To ensure robust generalization without false positives on human faces or household
items, we engineered a balanced 3-class dataset containing 9,000 total images
(3,000 images per class, split across 80% Train, 10% Validation, and 10% Test):

1. Cat (3,000 images):
   * Extracted from Google's filtered Cats vs. Dogs dataset (archive.zip).
   * Contains diverse breeds, lighting conditions, angles, indoor/outdoor scenes,
     and close-up facial features.

2. Dog (3,000 images):
   * Extracted from archive.zip.
   * Contains various breeds (puppies to large dogs), poses (running, sitting),
     and diverse backgrounds.

3. Other / Background (3,000 images):
   * Curated across 94 distinct real-world object categories from the Caltech-101
     benchmark (specifically excluding animal classes like dalmatian or leopard).
   * Explicitly includes:
     - Human Faces & Heads: Faces (facial structure, eyes, noses), Faces_easy, brain.
     - Vehicles & Transportation: airplanes, car_side, motorbikes, ferry, ketch.
     - Everyday Objects & Electronics: cellphone, laptop, camera, chair, lamp, watch.
     - Nature & Plants: bonsai, sunflower, butterfly, crab, dolphin, panda.
     - Buildings & Structures: pagoda, pyramid, stop_sign, windsor_chair.

Data Preprocessing & Augmentation:
  * Image Resizing: Standardized to 128x128 pixels x 3 channels (RGB).
  * Pixel Normalization: Rescaled to [-1.0, 1.0] (the mathematical expectation
    of MobileNetV2).
  * Real-Time Augmentation (ImageDataGenerator): Applied random rotations (20 deg),
    width/height shifts (20%), shear/zoom (20%), and horizontal flips during
    training to make the neural network invariant to camera angles and orientation.


4. MODEL ARCHITECTURE & TRANSFER LEARNING (02_build_model.py)
--------------------------------------------------------------------------------
Rather than training a basic custom Convolutional Neural Network (CNN) from scratch
—which often plateaus at ~75-80% accuracy—we implemented Transfer Learning using
the state-of-the-art MobileNetV2 architecture:

[Input: 128x128x3 RGB Image]
          │
          ▼
[Rescaling Layer: Rescale(scale=2.0, offset=-1.0) -> Maps [0,1] to [-1, 1]]
          │
          ▼
[MobileNetV2 Feature Extractor: Pre-trained on 14 Million ImageNet Photos (Frozen)]
  ├── Uses Depthwise Separable Convolutions for ultra-high mobile execution speed
  └── Outputs deep semantic texture maps (eyes, fur patterns, geometric edges)
          │
          ▼
[GlobalAveragePooling2D: Condenses feature maps into a 1D vector of 1,280 numbers]
          │
          ▼
[Dense Head: 256 Neurons + ReLU Activation]
          │
          ▼
[Dropout Layer: 50% Random Neuron Deactivation during training to prevent overfitting]
          │
          ▼
[Output Layer: Dense(3 Neurons + Softmax Activation) -> [P(Cat), P(Dog), P(Other)]]


5. TRAINING, EVALUATION & CONVERSION PIPELINE
--------------------------------------------------------------------------------
Training (03_train_model.py):
  * Optimizer: Adam with adaptive learning rate.
  * Loss Function: Categorical Crossentropy.
  * Callbacks:
    - EarlyStopping(patience=5): Prevents overfitting if validation loss stops improving.
    - ModelCheckpoint: Automatically saves the best-performing weights to
      models/best_model.keras.
  * Results: Reached 97.8%+ validation accuracy across all 3 classes in 5 epochs.

Evaluation (04_evaluate_model.py):
  * Generates classification reports (Precision, Recall, F1-score) and exports a
    visual Confusion Matrix (models/confusion_matrix.png) verifying minimal
    cross-class confusion between Cats, Dogs, and Background objects.

Edge Conversion (05_convert_tflite.py):
  * Converts the heavy Keras model (9.23 MB) into an ultra-compact TensorFlow Lite
    flatbuffer (model.tflite = 2.42 MB - 73.8% smaller!).
  * Patches flatbuffer opcodes to Version 9 to guarantee zero crashes on older
    Android hardware.
  * Automatically exports labels.txt containing ["cat", "dog", "other"].


6. ANDROID MOBILE APPLICATION ARCHITECTURE (com.example.imageclassifier)
--------------------------------------------------------------------------------
The Inference Wrapper (ImageClassifier.kt):
  1. Model Loading: Reads model.tflite and labels.txt from app/src/main/assets/
     using org.tensorflow.lite.Interpreter.
  2. Pre-Processing: Takes any input Bitmap (from Camera or Gallery), downscales
     it smoothly to 128x128 using bilinear filtering, and packs normalized floating
     point numbers into a ByteBuffer (196,608 bytes total).
  3. Execution: Runs interpreter.run(inputBuffer, outputArray) in milliseconds
     on the mobile CPU (accelerated via XNNPACK delegate).
  4. Output Extraction: Returns Pair(className, confidenceScore) e.g., Pair("Cat", 0.994f).

The User Interface (MainActivity.kt & activity_main.xml):
  * UI Components: A clean, centered Material Design CardView displaying the selected
    image, adjoined by two interactive action buttons (Take Photo & Pick from Gallery).
  * Intelligent Result Display: Intercepts raw predictions and presents human-readable
    results on screen:
    - If predicted class is "Cat" -> Displays: Cat
    - If predicted class is "Dog" -> Displays: Dog
    - If predicted class is "Other" -> Displays: Other (Not Cat/Dog)


7. KEY ENGINEERING CHALLENGES & LESSONS LEARNED
--------------------------------------------------------------------------------
The "Human Face as Dog" Phenomenon (Why Option B beat Option A):
During iterative testing, we explored two distinct strategies for handling
out-of-scope images (such as human faces):

* Option A (2-Class Softmax + Percentage Thresholding):
  We trained a 2-class model (Cat vs. Dog) and added an 85% confidence threshold check
  in Kotlin (if confidence < 0.85f then display "Unknown").
  - The Problem: When shown a human face, the model's pre-trained nose/eye feature
    maps resembled a dog's snout slightly more than cat whiskers. Because softmax
    forces two relative probabilities to sum to 100%, even a modest preference for
    Dog got exponentially amplified (e^z) into 93% false confidence, bypassing our check!
    When we raised the threshold to 99.5% to catch the face, it accidentally blocked
    real cat photos (which scored 99.28%).

* Option B (3-Class Architecture - The Solution):
  By adding the 3rd "Other" class trained on Caltech-101 faces, cars, and furniture,
  the neural network explicitly learned the visual structure of human faces and
  everyday objects. When shown a human face today, the model outputs "Other: 99.8%"
  natively without requiring fragile percentage limits!

How to Re-Run / Re-Deploy from Scratch:
--------------------------------------------------------------------------------
Whenever you add new images to your dataset or tweak hyperparameters, execute this
single terminal pipeline from the project root (C:\image_classification):

# 1. Train model & convert to TFLite
python src/03_train_model.py
python src/05_convert_tflite.py

# 2. Copy updated model & labels to Android Assets
Copy-Item -Force "models\model.tflite" "android-app\app\src\main\assets\model.tflite"
Copy-Item -Force "models\labels.txt" "android-app\app\src\main\assets\labels.txt"

# 3. Build APK & Stream Install to USB Device
cd android-app
.\gradlew.bat assembleDebug
cd ..
adb -s 10BE7C1J1G000E3 install -r "android-app\app\build\outputs\apk\debug\app-debug.apk"
